1. Обзор системы

Комплексная система оптимизации зарплаты на основе данных с автоматическим распределением

1.1 Цель алгоритма

  • 4-компонентная модель зарплаты с приоритизацией данных
  • Приоритизация Delta₁ (KPI) на основе исторической производительности
  • Автоматическое распределение ресурсов на Delta₂ и challenges
Total_Salary = Base_Salary + Delta₁ + Delta₂ + Challenge_Bonus

Base_Salary

Фиксированная гарантированная часть

Delta₁ (ПЕРВИЧНО)

Бонусы за KPI: телефоны, SIM, аксессуары

Delta₂ (ВТОРИЧНО)

Комиссии за сложные сделки

Challenge_Bonus

Мотивационные геймифицированные бонусы

1.3 Сравнение подходов

Аспект Традиционная Delta₁-Priority
Решение Выбор распределения (50/50, 70/30, 30/70) Только ввод желаемой зарплаты
Оптимизация Риск неоптимального выбора Data-driven автоматика
Время ~70 секунд ~25 секунд (-64%)
Вовлечение менеджера Нет Только для специалистов (≥80%)

1.4 Решаемая проблема

  • Индивидуальные вариации в способностях KPI между сотрудниками
  • Разделение между рутинными продажами (Delta₁) и сложными сделками (Delta₂)
  • Естественная специализация сотрудников (некоторые лучше в объемах, другие в сделках)
  • Временные ограничения и предотвращение выгорания
  • Контроль менеджера только для крайних случаев (≥80% специализация)

2.2 Процесс постановки цели (Delta₁-Priority)

Шаг 1: Ввод сотрудника

Ввод: "Хочу заработать $5,500 в этом месяце" Известно: Base_Salary = $2,200 (из HR) Расчет: Variable_Needed = $5,500 - $2,200 = $3,300

Шаг 2: Система рассчитывает максимальную емкость Delta₁

Вместо запроса выбора распределения система определяет:

  1. Какую максимальную производительность KPI может достичь сотрудник?
  2. Сколько бонуса Delta₁ это генерирует?
  3. Что остается для Delta₂ + Challenges?

2.3 Расчет емкости Delta₁ (Улучшенная версия)

Цель: Рассчитать максимальные достижимые цели KPI для сотрудника

Этап 1: Анализ исторической согласованности (для каждого KPI)

Для каждого KPI (телефоны, SIM, аксессуары): 1. Собрать историю за 6 месяцев: [Month₁, Month₂, Month₃, Month₄, Month₅, Month₆] 2. Рассчитать статистику: Average = Σ(Monthly_Values) / 6 Maximum = MAX(Monthly_Values) Minimum = MIN(Monthly_Values) Range = Maximum - Minimum 3. Рассчитать Consistency Score: Consistency = 1 - (Range / Maximum) 4. Классифицировать: IF Consistency ≥ 0.70: Level = HIGH ELSE IF Consistency ≥ 0.40: Level = MEDIUM ELSE: Level = LOW
Пример — Джон (Телефоны):
История: [45, 47, 46, 44, 48, 45] Среднее = 45.8 Maximum = 48, Minimum = 44 Range = 4 Consistency = 1 - (4/48) = 0.917 (HIGH)

Этап 2: Назначение весов доверия

IF Consistency = HIGH: Trust_Personal = 0.90 Push_Factor = 1.05 - 1.08 (переменный) ELSE IF Consistency = MEDIUM: Trust_Personal = 0.70 Push_Factor = 1.02 ELSE (LOW): Trust_Personal = 0.50 Push_Factor = 1.00 (без push)

Этап 3: Расчет смешанной цели

Base_Target = (Employee_Average × Trust_Personal) + (Company_Average × (1 - Trust_Personal))
Пример — Джон (Телефоны):
Employee_Average = 45.8 Company_Average = 50 Trust_Personal = 0.90 Base = (45.8 × 0.90) + (50 × 0.10) = 46.22

Этап 4: Применение Push Factor

Variable_Push = Push_Base + ((Consistency - 0.70) × Push_Range / 0.30) Для Джона (Consistency = 0.917): Variable_Push = 1.05 + ((0.917 - 0.70) × 0.03 / 0.30) = 1.072 Pushed_Target = 46.22 × 1.072 = 49.55 телефонов

Этап 5: Ограничения безопасности

Максимальная граница: (максимум из истории * 1.1) Минимальный порог: (среднее из истории * 0.9)

Этап 6: Расчет максимальной достижимой Delta₁

Структура бонусов KPI:
Каждый KPI имеет уровни достижения: <90%: множитель 0× 90-99%: пропорционально 100-104%: множитель 1.0× 105-109%: множитель 1.1× 110-119%: множитель 1.2× ≥120%: множитель 1.5× Базовые значения бонусов (при 100%): Телефоны: $600 SIM-карты: $500 Аксессуары: $300

2.4 Автоматическое распределение Delta₂ + Challenge (НОВОЕ)

⚠️ Ключевой момент: Распределить оставшийся переменный доход на Delta₂ и Challenges на основе доступной емкости

Шаг 1: Рассчитать оставшийся переменный бюджет

Variable_Needed = Total_Goal - Base_Salary Realistic_Delta₁ = (из расчета Проблемы 2.3) Remaining_Budget = Variable_Needed - Realistic_Delta₁ Пример: Variable_Needed = $3,300 Realistic_Delta₁ = $1,365 Remaining_Budget = $3,300 - $1,365 = $1,935

Шаг 2: Рассчитать процент Delta₁

Delta₁_Percentage = Realistic_Delta₁ / Variable_Needed Пример: Delta₁_Percentage = $1,365 / $3,300 = 0.414 (41.4%)

Шаг 3: Дерево автоматического распределения

IF Delta₁_Percentage < 0.50: // Сотрудник имеет высокую емкость для сделок Delta₂_Allocation = Remaining_Budget × 0.75 Challenge_Allocation = Remaining_Budget × 0.25 Status = "Сбалансированный - Высокая емкость для сделок" Manager_Approval = НЕ ТРЕБУЕТСЯ ELSE IF Delta₁_Percentage >= 0.50 AND Delta₁_Percentage < 0.70: // Стандартный сбалансированный исполнитель Delta₂_Allocation = Remaining_Budget × 0.70 Challenge_Allocation = Remaining_Budget × 0.30 Status = "Сбалансированный - Стандартный микс" Manager_Approval = НЕ ТРЕБУЕТСЯ ELSE IF Delta₁_Percentage >= 0.70 AND Delta₁_Percentage < 0.80: // Исполнитель с уклоном в KPI Delta₂_Allocation = Remaining_Budget × 0.60 Challenge_Allocation = Remaining_Budget × 0.40 Status = "KPI-Ориентированный" Manager_Approval = НЕ ТРЕБУЕТСЯ ELSE IF Delta₁_Percentage >= 0.80: // Тяжелый специалист по KPI - требуется проверка Delta₂_Allocation = Remaining_Budget × 0.50 Challenge_Allocation = Remaining_Budget × 0.50 Status = "KPI Специалист - Требуется проверка менеджера" Manager_Approval = ТРЕБУЕТСЯ Alert_Message = "Емкость KPI сотрудника составляет 80%+ от переменного дохода. Только 20% доступно для сделок. Рассмотрите: 1. Принять (сотрудник — специалист по KPI) 2. Уменьшить общую цель для баланса 3. Пользовательское переопределение"
Пример — Джон (Delta₁ = 41.4%):
Delta₁_Percentage = 41.4% → Попадает в диапазон <50% Delta₂_Allocation = $1,935 × 0.75 = $1,451 Challenge_Allocation = $1,935 × 0.25 = $484 Итоговое распределение: Delta₁: $1,365 (41.4%) Delta₂: $1,451 (44.0%) Challenges: $484 (14.6%) Всего переменная: $3,300
Пример — Тяжелый специалист по KPI (Delta₁ = 85%):
Variable_Needed = $3,000 Realistic_Delta₁ = $2,550 (85%) Remaining = $450 ⚠️ Система сигнализирует: "Требуется одобрение менеджера" Delta₂_Allocation = $450 × 0.50 = $225 Challenge_Allocation = $450 × 0.50 = $225 Менеджер видит: "Емкость KPI сотрудника Sarah: $2,550 (85% от переменной) Только $450 (15%) доступно для сделок и challenges. Sarah может быть чистым специалистом по KPI. Варианты: 1. ✓ Принять (Sarah фокусируется на объемных продажах) 2. Уменьшить цель до $4,200 (балансирует до 75/25) 3. Пользовательское: Установить Delta₂ на $500, Delta₁ на $2,500"

2.5 Расчет целей Delta₂

Цель: Преобразовать выделенный бюджет Delta₂ в реалистичные цели по сделкам

Шаг 1: Исторический анализ сделок

Собрать историю сделок за 6 месяцев: - Всего закрытых сделок - Микс сделок по уровням (% Tier 1, 2, 3, 4, 5) - Средняя комиссия на сделку - Коэффициент закрытия Пример — Джон: Последние 6 месяцев: 21 сделка закрыта, 32 попытки Close_Rate = 21/32 = 65.6% Avg_Commission_Per_Deal = $3,869 / 21 = $184 Deal_Mix: Tier1=19%, Tier2=43%, Tier3=29%, Tier4=9%, Tier5=0%

Шаг 2: Расчет целевых сделок

Allocated_Delta₂ = $1,451 (из автоматического распределения) Deals_Needed = Allocated_Delta₂ / Avg_Commission_Per_Deal = $1,451 / $184 = 7.9 ≈ 8 сделок Корректировка на коэффициент закрытия: Deals_To_Attempt = 8 / 0.656 = 12.2 ≈ 12 попыток сделок Распределение по историческому миксу: Tier 1 (19%): 12 × 0.19 = 2.3 ≈ 2 простых пакета Tier 2 (43%): 12 × 0.43 = 5.2 ≈ 5 семейных планов Tier 3 (29%): 12 × 0.29 = 3.5 ≈ 4 trade-in Tier 4 (9%): 12 × 0.09 = 1.1 ≈ 1 малый бизнес

2.6 Распределение Challenge (Автоматическое)

Шаг 1: Проверка права на участие

IF Current_Performance < 80% базовых целей: Eligible = FALSE Challenge_Allocation = 0 (перенаправляется на Delta₂) ELSE: Eligible = TRUE

Шаг 2: Распределение бюджета Challenge

Allocated_Challenge_Budget = $484 (из автоматического распределения) Система генерирует challenges стоимостью примерно этой суммы: - Challenge 1: $150 (Volume Sprint) - Challenge 2: $200 (Consistency Streak) - Challenge 3: $150 (Deal Mastery) Всего предложено: $500 (немного выше бюджета для выбора) Сотрудник выбирает до 2-3 challenges общей стоимостью ~$484
Типы Challenge:
  1. Volume Sprint: Продать X единиц за 7 дней
  2. Consistency Streak: Достичь цели N дней подряд
  3. Deal Mastery: Закрыть N сделок Tier X+
  4. Product Mix: Достичь всех целей KPI в одну неделю
  5. Team Collaborative: Командная цель

3. ПОД-АЛГОРИТМ 1: ВАЛИДАЦИЯ ВРЕМЕНИ

Цель: Проверить, что комбинированные цели Delta₁ + Delta₂ укладываются в доступное рабочее время

Effective_Selling_Time = (Full_Day - Non_Selling_Activities) × Safety_Buffer = (480 - 140) × 0.85 = 289 минут/день Total_Time_Required = KPI_Time + Deal_Time KPI_Time = Σ(KPI_Quantity_i × Time_Per_Unit_i) Deal_Time = Σ(Deal_Target_j × Time_Per_Tier_j) IF Total_Time_Required ≤ 289: ВЫПОЛНИМО ELSE: НЕВЫПОЛНИМО → Применить пропорциональное сокращение
Пример — Джон:
Время KPI: - 2.3 телефона × 18 мин = 41.4 мин - 2.9 SIM × 7 мин = 20.3 мин - 1.1 аксессуара × 10 мин = 11 мин Промежуточный итог: 72.7 мин Время сделок (усредненно в день): - 0.1 Tier 1 × 45 мин = 4.5 мин - 0.25 Tier 2 × 90 мин = 22.5 мин - 0.2 Tier 3 × 90 мин = 18 мин - 0.05 Tier 4 × 240 мин = 12 мин Промежуточный итог: 57 мин Всего: 129.7 минут < 289 ✓ ВЫПОЛНИМО Буфер: 159.3 минуты (55% запас)
Интеграция с распределением:
  • Если невыполнимо, уменьшить ОБЕИ Delta₁ и Delta₂ пропорционально
  • Пересчитать достижимую цель зарплаты
  • Представить скорректированную цель сотруднику

4. ПОД-АЛГОРИТМ 2: РАСПРЕДЕЛЕНИЕ ЗАПАСОВ

Цель: Распределить ограниченные запасы на основе недавней производительности

Взвешенная оценка производительности

Score = (W₁×S₁ + W₂×S₂ + W₃×S₃ + W₄×S₄) / 7.0 Где: W = [1.0, 1.5, 2.0, 2.5] для недель 1-4 (Более свежие недели имеют больший вес)

Классификация Активный/Неактивный

IF Calculator_Last_Used ≤ 5 дней: Status = АКТИВНЫЙ (100% распределение) ELSE: Status = НЕАКТИВНЫЙ (70% базовое распределение)

Распределение по заслугам

Employee_Allocation = (Employee_Score / Total_Active_Score) × Available_Stock
Еженедельная перекалибровка

Дни 1, 6, 11, 16: Пересчитать оценки, используя только данные текущего месяца

Примечание: Изменения не требуются — распределение запасов не зависит от подхода Delta₁-priority.

5. ПОД-АЛГОРИТМ 3: СИСТЕМА МОМЕНТА 5 ПОЛОС

Цель: Динамически корректировать цели KPI Delta₁ на основе трехдневных паттернов производительности

Классификация полос:

Полоса Ratio Статус Баллы
Полоса 1 ≥ 1.15 Разбивая! +2 балла
Полоса 2 1.05-1.14 Превышающий +1 балл
Полоса 3 0.95-1.04 На пути 0 баллов
Полоса 4 0.85-0.94 Позади -1 балл
Полоса 5 < 0.85 Борющийся -2 балла

Расчет момента:

Требуется серия из 3 дней: [Day₋₂, Day₋₁, Day₀] Momentum_Total = Сумма баллов полос Факторы корректировки: ≥5: ×1.10 (+10%) 3-4: ×1.05 (+5%) -2 to +2: ×1.00 (поддерживать) -4 to -3: ×0.92 (-8%) ≤-5: ×0.85 (-15%) Ограничения безопасности: Макс ±15% в день
⚠️ Важно: Применяется ТОЛЬКО к Delta₁ (KPIs) — сделки Delta₂ отслеживаются отдельно в pipeline

6. ПОД-АЛГОРИТМ 4: СИСТЕМА КОНТРОЛЬНЫХ ТОЧЕК

Цель: Стратегические оценки вех в Дни 6, 10, 16, 20 по ВСЕМ компонентам

Расчет комплексного соотношения (ОБНОВЛЕНО)

Variable_Target = Delta₁_Target + Delta₂_Target + Challenge_Bonuses Expected_Progress = Variable_Target × Expected_Percentage Actual_Progress = Delta₁_Earned + Delta₂_Earned + Challenges_Completed Ratio = Actual_Progress / Expected_Progress

Компонент-специфичные корректировки (НОВАЯ ЯСНОСТЬ)

При применении корректировок контрольных точек система использует интеллектуальное распределение:

IF Delta₁ на пути НО Delta₂ позади: Применить БОЛЬШЕ сокращения к Delta₂ Применить МЕНЬШЕ сокращения к Delta₁ IF Delta₂ на пути НО Delta₁ позади: Применить БОЛЬШЕ сокращения к Delta₁ Применить МЕНЬШЕ сокращения к Delta₂ IF оба позади одинаково: Применить пропорциональное сокращение к обоим
Пример корректировки:
Контрольная точка Дня 10 показывает 82% комбинированное (требуется корректировка ×0.92) Анализ компонентов: Delta₁: $650/$650 ожидается = 100% ✓ Delta₂: $300/$500 ожидается = 60% ⚠️ Challenges: $0/$150 ожидается = 0% Умная корректировка: - Delta₁: Поддерживать на $1,365 (хорошо выполняется) - Delta₂: Уменьшить на 20% → $1,451 × 0.80 = $1,161 - Challenges: Убрать 1 challenge → $484 → $334 Новые цели отражают реальность: Сотрудник превосходит в KPI, борется со сделками

7. ПОД-АЛГОРИТМ 5: СИСТЕМА ОПОВЕЩЕНИЙ В РЕАЛЬНОМ ВРЕМЕНИ

Цель: Ежедневное прогнозирование траектории цели с компонент-специфичной диагностикой

Мультикомпонентное прогнозирование

Delta₁_Pace = Cumulative_Delta₁ / Days_Elapsed Delta₁_Projected = Delta₁_Pace × 20 Delta₁_Ratio = Delta₁_Projected / Delta₁_Target Delta₂_Pace = Cumulative_Delta₂ / Days_Elapsed Delta₂_Projected = Delta₂_Pace × 20 Delta₂_Ratio = Delta₂_Projected / Delta₂_Target Challenges_Projected = Completed + (Active_Likely × 0.5) Combined_Projected = Delta₁_Proj + Delta₂_Proj + Challenge_Proj Combined_Ratio = Combined_Projected / Variable_Target

Определение уровня оповещения

IF (Delta₁_Ratio < 0.70 ИЛИ Delta₂_Ratio < 0.70) И Day ≥ 8: Level = КРИТИЧЕСКИЙ Primary_Issue = Компонент с более низким соотношением ELSE IF (Delta₁_Ratio < 0.85 ИЛИ Delta₂_Ratio < 0.85) И Day ≥ 6: Level = ПРЕДУПРЕЖДЕНИЕ Primary_Issue = Компонент с более низким соотношением ELSE IF Combined_Ratio < 0.94 И Day ≥ 5: Level = ОСТОРОЖНОСТЬ ELSE: Без оповещения

Умное дросселирование

  • Правило 1: Макс 3 оповещения в неделю (5 рабочих дней)
  • Правило 2: Не одно и то же оповещение уровня в течение 2 дней (кулдаун)
  • Правило 3: Эскалация всегда разрешена (ОСТОРОЖНОСТЬ→ПРЕДУПРЕЖДЕНИЕ→КРИТИЧЕСКИЙ)

Компонент-специфичные планы восстановления

Восстановление Delta₁:
Remaining_Needed = Delta₁_Target - Delta₁_Earned Required_Daily = Remaining_Needed / Days_Left Current_Daily = Delta₁_Earned / Days_Elapsed Increase_Needed = (Required_Daily / Current_Daily - 1) × 100% Преобразовать в единицы: "Увеличить телефоны с 2.1/день до 3.2/день"
Восстановление Delta₂:
Deals_Needed = (Delta₂_Target - Delta₂_Earned) / Avg_Commission Deals_To_Attempt = Deals_Needed / Close_Rate Days_Per_Deal = Days_Left / Deals_To_Attempt Сообщение: "Закрыть еще 4 сделки за 9 дней = 1 сделка каждые 2.25 дня"

8. СИСТЕМА CHALLENGE

Цель: Геймифицировать исключительную производительность с добровольными целями растяжения

Типы Challenge:

  1. Volume Sprint: Продать X единиц за 7 дней
  2. Consistency Streak: Достичь цели N последовательных дней
  3. Deal Mastery: Закрыть N сделок Tier X+
  4. Product Mix: Достичь всех целей KPI в одну неделю
  5. Team Collaborative: Командная цель

НОВОЕ: Интеграция с автоматическим распределением

Бюджет challenge автоматически определяется (обычно 10-30% от оставшейся переменной после Delta₁):

IF Delta₁_Percentage < 50%: Challenge_Allocation = Remaining × 0.25 (больше емкости) ELSE IF Delta₁_Percentage < 70%: Challenge_Allocation = Remaining × 0.30 (стандарт) ELSE IF Delta₁_Percentage < 80%: Challenge_Allocation = Remaining × 0.40 (компенсировать тяжелый фокус на KPI) ELSE: Challenge_Allocation = Remaining × 0.50 (случай одобрения менеджера)
